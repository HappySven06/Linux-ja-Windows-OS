# 20.04 Tunni ülesanne – Active Directory

**Eesmärk:** Õppida seadistama Active Directory (AD) Windows Serveris ja ühendama Windows klienteid domeeni.

## Ülesande kirjeldus

Teie ülesanne on seadistada Active Directory (AD) Windows Serveris TPTLab keskkonnas ning loo domeen. Seejärel tuleb luua AD kasutaja ja ühendada Windows kliendimasin domeeniga, et testida AD funktsionaalsust.

## Ülesande sammud

1. **Seadista Active Directory Windows Serveris**
   - Seadista AD serverile staatiline IP-aadress
   - Seadista Windows Server Domain Controlleri rolliks
     - Paigalda Active Directory Domain Services (AD DS)
     - Tõsta serverit domeeni kontrolleriks
   - Loo domeen
     - Domeen nimi: **"sinu eesnimi".lab.local** (näiteks: `jhon.lab.local`)
     - Veendu, et domeeni leht on õigesti seadistatud

2. **Loo kasutaja Active Directorys**
   - Ühenda AD haldusvahendiga (Active Directory Users and Computers)
   - Loo uus kasutaja
     - Kasutaja nimi: `teacher`
     - Parool: `Teacher1`
     - Veendu, et parool vastab komplekssusnõuetele
     - Lisa kasutaja sobivasse organisatsioonilisele ühikusse (OU)

3. **Seadista Windows kliendimasin**
   - Loo Windows 10/11 VM
   - Ühenda kliendimasin AD domeeeniga
     - Lisa arvuti domeeni (sinu.lab.local)
     - Veendu, et ühendamine õnnestus

4. **Testa AD funktsionaalsust**
   - Logi kliendimasinasse sisse `teacher` kasutajaga
     - Veendu, et kuvatakse domeenikasutaja sisselogimine
     - Kontrolli, et kasutaja profii on õigesti seadistatud 


## Täiendavad nõuanded

- Veendu, et Windows Server on korrektselt seadistatud enne AD paigaldamist.
- Dokumenteeri domeeni nimi ja administraatori paroolid turvaliselt.
- Kui domeeni liitumine ebaõnnestub, kontrolli DNS ja võrguühendust.
- Testi enne lõplik valideerimist, et kõik komponendid omavahel suhtlevad.
- Active Directory Admin Center on mugavam kasutajate haldusvahend kui vana ADU&C tööriist.
